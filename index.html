Cobble a responsive grid system with a difference

This is a demonstration of a responsive grid system using inline-block columns instead of floating ones. It also makes use of HTML5 semantic elements for easier reading.

The system otherwise borrows from the popular responsive framework, Twitter Bootstrap. Namely the container — row — column system and the container width breakpoints. It also supports the 12 column grid as this provides the most flexible page divisibility

This example uses the HTML5 Section element as the containing div. The container class is "box".

The "row" class can be applied to any HTML block level element, just like it can in Bootstrap. The columns are called "panel". In this case I have applied the column classes to the Article and Aside elements.

To create a column, use the class 'panel' combined with the class 'panel-*', where '*' is the number of columns you want the panel to span.

The same coding rules apply to the classes here as to Bootstrap classes: Rows must be placed inside containers; columns inside rows; rows containing other columns can be nested inside columns; and containers must not be placed inside containers.

I have also used the Flex display property on the Row class at medium and large screen sizes, which means that columns with less content than their siblings will still extend vertically to fill the row when arranged inline. When stacked, the rows become block elements. This eliminates the need for a clearfix.

This grid system is designed to be 'mobile-first', meaning that the grid CSS foundations have been designed to fit the smallest screen sizes by default and the media queries adapt the layout for larger screens when necessary.
